## 🔍 التحليل الفني:

### المشكلة المكتشفة:

#### 1. carx-system لا يستخدم Multi-Tenant API
```typescript
// في carx-system/src/lib/TenantContext.tsx
const defaultTenant = {
  id: 'carx',
  name: 'CAR X',
  // ... ثابت في الكود
};

// ❌ لا يستدعي /api/v2/tenant/info
// ❌ لا يرسل X-Tenant-ID header
```

#### 2. carx-system له قاعدة بيانات خاصة في .env.local
```env
MONGO_URI=mongodb+srv://.../carx_production
```

**لكن:** هذا المتغير لا يُستخدم! لأن carx-system هو Frontend فقط.

#### 3. Backend API (daood.okigo.net) يحدد العميل من Domain
```javascript
// في tenant-resolver.js
const host = req.headers.host;

if (host === 'daood.okigo.net') → hmcar
if (host === 'carx-system-psi.vercel.app') → carx
```

**المشكلة:** carx-system يستدعي daood.okigo.net/api
→ Backend يرى host = daood.okigo.net
→ يرجع بيانات hmcar ❌

## 🎯 الحل الأمثل:

### الخيار 1: إضافة X-Tenant-ID Header (الأسرع)

**الفكرة:** carx-system يرسل header مع كل طلب API

**التعديل المطلوب:**

#### 1. تعديل carx-system/src/lib/api.ts
```typescript
private async request<T>(
  endpoint: string,
  options: RequestInit = {}
): Promise<ApiResponse<T>> {
  try {
    const url = `${this.baseUrl}${endpoint}`;
    const response = await fetch(url, {
      headers: {
        'Content-Type': 'application/json',
        'X-Tenant-ID': 'carx',  // ← إضافة هذا السطر
        ...options.headers,
      },
      ...options,
    });
    // ...
  }
}
```

**المميزات:**
- ✅ سريع (تعديل ملف واحد)
- ✅ يعمل فوراً
- ✅ لا يحتاج تغيير Backend

**العيوب:**
- ❌ يحتاج Redeploy لـ carx-system
