# خطوات النشر الآمن لـ HM CAR

// turbo-all

## قبل النشر - فحص الكود

### 1. فحص البناء بدون نشر (اختياري)

```powershell
node deploy.js --check
```

> إذا نجح: تابع. إذا فشل: أصلح الأخطاء أولاً.

## النشر الكامل

### 2. تشغيل سكريبت النشر الذكي

```powershell
cd c:\car-auction
node deploy.js
```

هذا السكريبت يقوم تلقائياً بـ:

- ✅ فحص الملفات الأساسية
- ✅ تحديث Service Worker (لكسر الكاش القديم)
- ✅ بناء المشروع (`next build`)
- ✅ إذا فشل البناء: **يتوقف** ولا ينشر
- ✅ حفظ Git Commit
- ✅ نشر على Vercel Production

## بعد النشر

### 3. تحقق من ظهور التحديثات

- افتح الموقع على متصفح **Incognito** (نافذة خاصة)
- أو اضغط **Ctrl+Shift+R** (Hard Refresh)
- التغييرات يجب أن تظهر **فوراً**

## في حالة مشاكل الكاش

### 4. إذا لم يظهر التحديث للعميل

```powershell
# أعد النشر بإصدار SW جديد يكسر كل الكاشات
node deploy.js
```

### 5. فحص Vercel Logs للمشاكل

```powershell
vercel logs --prod
```

## ملاحظات مهمة

- **البيانات (MongoDB Atlas)** لا تتأثر أبداً بالنشر
- **الصور (Cloudinary)** لا تتأثر أبداً بالنشر
- **إعدادات Vercel** (Environment Variables) يجب ضبطها مرة واحدة فقط
- كل نشر يولّد **رقم إصدار جديد** للـ Service Worker تلقائياً
