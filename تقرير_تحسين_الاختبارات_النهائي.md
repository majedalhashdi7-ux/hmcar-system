### 1. إصلاح Password Hashing ✅

**المشكلة:**
- المستخدمون يتم إنشاؤهم بـ `User.create()` مع password مشفر مسبقاً
- الـ `pre('save')` hook في User model يقوم بتشفير الـ password مرة أخرى
- النتيجة: double hashing يجعل المقارنة تفشل

**الحل:**
```javascript
// ❌ خطأ - double hashing

    ...userData,
    password: await hashPassword(userData.password),
});

// ✅ صحيح - single hashing via hook
await User.create(userData);
```

### 2. تحسين Order Validation ✅

**المشكلة:**
- Order items تحتاج حقول إضافية: `titleSnapshot`, `refId`, `itemType`
- `createOrderData()` لا يضيف هذه الحقول تلقائياً

**الحل:**
```javascript
function createOrderData(userId, items, overrides = {}) {
    const mongoose = require('mongoose');
    
    // Process items to add required fields
    const processedItems = items && items.length > 0 
        ? items.map(item => ({
            titleSnapshot: item.titleSnapshot || 'Test Item',
            refId: item.refId || item.part || item.car || new mongoose.Types.ObjectId(),
            itemType: item.itemType || (item.part ? 'part' : 'car'),
            price: item.price || 100,
            quantity: item.quantity || 1,
            ...item
        }))
        : [];
    
    return {
        buyer: userId,
        items: processedItems,
        orderNumber: `HM-2026-${Math.floor(Math.random() * 1000000).toString().padStart(6, '0')}`,
        status: 'pending',
        ...overrides,
    };
}
```
