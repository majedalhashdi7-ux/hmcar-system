# 🔄 دليل التحديث السريع والآمن (عبر Git)
# ════════════════════════════════════════

هذا الدليل مخصص لك (كمدير للسيرفر) لجلب التحديثات الجديدة للنظام فور انتهاء المطور منها.
بدلاً من رفع ملفات ZIP كل مرة، نستخدم نظام Git لضمان الاحترافية وسرعة تحديث النظام بدون توقف.

## 📥 كيف يتم السحب لأول مرة؟
إذا لم تقم بتجهيز المجلد على السيرفر بعد، اكتب:
```bash
cd /var/www
git clone https://github.com/YOUR_GITHUB_USERNAME/hmcar-system.git hmcar
cd hmcar
```

## 🚀 كيفية التحديث اليومي أو عند أي تعديل؟
عندما يُخبرك المطور أنه تم إصدار تحديث جديد ويطلب منك "تحديث السيرفر"، افتح الـ Terminal (SSH) ونفذ هذه الأسطر الخمسة فقط بالترتيب:

```bash
cd /var/www/hmcar

# 1. سحب آخر التحديثات والأكواد الجديدة
git pull origin main

# 2. تحديث مكتبات الباك إند (احتياطاً)
npm install --production

# 3. تحديث مكتبات الفرونت وإعادة البناء
cd client-app 
npm install 
npm run build 
cd ..

# 4. إعادة تشغيل النظام بالكامل ليقرأ التحديثات
pm2 restart all
```

## ⚠️ مشاكل شائعة وحلولها السريعة:
- **في حالة ظهر لك تعارض (Conflict) عند `git pull`:** 
هذا يعني أن ملفات السيرفر تعدلت يدوياً. لحلها وإجبار السيرفر على أخذ تحديثات المطور، اكتب:
`git fetch --all && git reset --hard origin/main` ثم كرر خطوات البناء.

- **لمعرفة هل النظام قيد العمل حالياً:** 
اكتب `pm2 status`.

- **لمشاهدة أي أخطاء مباشرة وقت التشغيل:**
اكتب `pm2 logs hmcar-api` (للباك إند) أو `pm2 logs hmcar-web` (للفرونت).
