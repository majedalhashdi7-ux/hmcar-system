# HM CAR - تقرير الجاهزية والنقل الفني (Technical Readiness Report) 🚀

هذا المستند موجه للفريق التقني المسؤول عن النشر والصيانة، ويلخص الحالة الحالية للنظام والخطوات المطلوبة للانتقال إلى مرحلة الإنتاج.

## 1. البنية التحتية المحدثة (Unified Hubs)

تم إعادة تنظيم لوحة التحكم لتعتمد نظام "المراكز الموحدة" لتقليل التشتت:

- **Terminal Hubs**: دمج الطلبات، المحادثات، والمزادات في واجهات مركزية.
- **Command Deck**: إضافة نظام بحث ذكي (Ctrl+K) للوصول السريع للأوامر.
- **Diagnostics Hub**: نظام مراقبة حي لفحص صحة الـ API، قاعدة البيانات، وسلامة الصور، ومنطق التسعير المالي (FINANCIAL LOGIC).

## 2. الوضع الأمني الحالي (Current Security State)

لقد تم إجراء "تطهير شامل" (Sanitization) للمستودع:

- **Environment Files**: تم إزالة جميع التوكنات (Vercel OIDC) وروابط قاعدة البيانات الحقيقية من المستودع.
- **Security Rotation**: تم استبدال الأسرار بـ Placeholders. يجب تدوير المفاتيح في لوحة تحكم Vercel/Railway.
- **Git Safety**: تم تحديث `.gitignore` لضمان عدم تتبع أي ملفات بيئة محلية مستقبلاً.

## 3. إرشادات النشر (Deployment Guidelines)

### استراتيجية النشر الهجين (Hybrid)

نظراً لوجود **WebSockets** و **Scrapers**، نوصي بالتالي:

- **Frontend (Vercel)**: يتم بناء تطبيق `client-app` ونشره.
- **Backend (VPS / Railway)**: يتم تشغيل الخادم الرئيسي لضمان استقرار الاتصالات المستمرة (Long Lived Connections).

### السكربتات المحدثة

- تم تحديث سكربت الـ `build` في الجذر ليقوم بالبناء الفعلي للواجهة.
- تم إضافة خطوة `npm run lint` في الـ CI لضمان استقرار الكود قبل الدمج.

## 4. قائمة التحقق النهائية (Final Checklist)

- [ ] إدخال `MONGO_URI` الحقيقي في إعدادات المنصة.
- [ ] ضبط `ALLOWED_ORIGINS` ليتناسب مع رابط الدومين الجديد.
- [ ] تفعيل `partsMultiplier` و `auctionMultiplier` من صفحة الإعدادات (قسم العملة).
- [ ] تدوير (Rotate) مفتاح `VERCEL_OIDC_TOKEN` في Vercel Dashboard.
